## 开房大数据查询
1. 获取文件数据
2. 数据清洗
3. 将数据入库
    1. 并发入库
4. 在终端循环输入通过姓名查询相关信息
    1. 实现精确查询/模糊查询(不加索引)
    2. 实现精确查询/模糊查询(添加索引,选做)
    3. 实现一个memory-mysql的二级缓存(必做)
        1. 自己实现缓存淘汰策略
        2. 缓存中允许存储的人名信息上限为10条
        3. 如果出现新的人名信息,将原来缓存中存在的数据删掉3条
        4. 将加入缓存最早的三条数据删除
        5. 策略2(选做),加入访问次数统计,在达到上限时,优先删除访问次数最少的三条
            如果说有访问次数相同的情况出现,再根据加入缓存的时间进行淘汰
    4. 实现redis-mysql的二级缓存查询策略(选做)
